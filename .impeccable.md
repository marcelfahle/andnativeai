# Design Context — &native.ai governed memory appliance

## Target audience
- SME founders / ops leaders evaluating a governed AI operator during a
  screen-share demo (prospects). They are smart but not infra people.
- Admin operators (Marcel, Matt) who run tenants, connect Slack, upload
  documents, and audit what the agent did.

## Use cases
- Answer in seconds: what does the agent know, where did an answer come from,
  what did the system just do, and who allowed it.
- Show trust at runtime: ingestion, retrieval, answering, deletion, and policy
  changes as inspectable evidence — not marketing claims.
- Operate: connect Slack, manage sources and per-channel policies, watch
  runtime health.

## Brand personality / tone
- Calm, precise, operational. Linear's density and keyboard-fast scanning;
  Vercel's restraint and typographic hierarchy. No enterprise-dashboard
  cosplay, no fake charts, no glow.
- Kathy Sierra lens: the UI's job is to make the operator feel capable.
  Evidence over decoration; next-step hints in empty states; labels that say
  what a thing means, not what widget it is.
- Honest data only: real counts get real numbers; estimates and placeholders
  are explicitly labeled as such.

## System notes
- Phoenix LiveView + Tailwind v4 + daisyUI tokens (base-100/200/300,
  base-content, primary, success/warning/error) with light + dark themes.
- Icons: heroicons via `<.icon name="hero-..." />` only.
- Type: system stack today; use `tabular-nums` for all counts/timestamps and
  mono for identifiers (request ids, channel ids).
- Motion: 150–250ms, ease-out, transform/opacity only, honor
  `prefers-reduced-motion`. No animation on keyboard-triggered actions.
